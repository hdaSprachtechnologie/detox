Schema Default
Table Annotations

Table that lists all annotations. It has one row per annotated comment from a specific annotator to a specific time.
The reson for this is, that every comment was annotated by multiple annotators and some comments were annotated twice by the same annotator, which makes it necassary to add the time in the primary key set.

Table Comments

Table with all existing (annotated and not annotated) comments.
Each row is a comment and the columns describe its properties. Not every comment has values for all properties as they partly depend on the source of a comment (see column source).

IdxField NameData TypeDescription
* c_id TEXT Unique ID of the comment.
* c_text TEXT The comments text.
  md5 TEXT md5-hash of the comment text (column c_text)
  date TIMESTAMP Timestamp (date and time) when the comment was posted on Twitter.
* source TEXT Describes if the comment is ether from the conversations part ("TW_Conv") or if it is a single comment ("TW_Com"). Some properties like "in_reply_to_user_id" depend on the source, that means only comments from conversations or single comments will have this attribute.
  context TEXT Column not in use anymore, is null for all comments.
  conv_id TEXT Conversation-ID of a comment. Only set if source="TW_Conv"
  author_id TEXT ID of the Twitter-user that posted the comment.
  in_reply_to_user_id TEXT To which user (user-ID) is this comment a response? - i.e. the user of the parent comment.
Only set if source="TW_Conv".
  like_count INTEGER Number of likes
  quote_count INTEGER Number of quotes.
  retweet_count INTEGER Number of retweets.
  reply_count INTEGER Number of of replies.
  c_type TEXT Only set if source="TW_Conv". Says if it is a main-comment ("main") or a comment in the conversation tree underneath a main-comment ("side"). Every conversation has exactly one main-comment which is the original post that stated the conversation. All other comments of the same conversation (same conversation-ID) are side-comments.
  preClassHate INTEGER Is true (=1) if the comment was prefiltered as probably offensive with our Hateword/Offensive-word-Lists. Null otherwise.

Table HateSpeechEntities

Table that saves the inputs of the 'Hate Speech Entities' - Field in the annotation schema.
Annotators should add words or phrases that were pivotal in their decision to label the comment as Hate Speech.
One annotation can contain multiple hate speech entites.

IdxField NameData TypeDescription
* c_id TEXT Comment-ID
* name TEXT Name of the annotator.
* time_response TIMESTAMP Time when the annotator finished annotating this comment.
* hate_speech_entity TEXT(12) Hate speech Entity that was enterd in the free text input.
Foreign Keys
FK_HateSpeechEntities Annotations (c_id, name, time_response) → Annotations


Table TW_References

Table that the references of comments from conversations and makes it therefore possible to reconstruct the conversation trees.
Every row lists which comment is referenced by the comment with the given c_id. Possible reference types are "quoted", "replied_to" or "retweeted".
One comment can have multiple references to other comments.

IdxField NameData TypeDescription
* c_id TEXT Comment-ID of a comment.
* ref_id TEXT The comment-ID of the referenced comment.
The referenced comment does not necessarily be available in the Comments-Table).
* ref_type TEXT Kind of reference. Possible are "quoted", "replied_to" and "retweeted"
Foreign Keys
FK_TW_References Comments (c_id) → Comments
Constraints
  ck_TW_References_ref_type ("ref_type" IN ('quoted', 'replied_to', 'retweeted'))


View Goldstandard

This table contains kind of a Goldstandard. It is almost the same like the view 'v_Voting' but with the difference, that comments that were re-annotated (checked) by the authors (annotators G1-3) are seen as correct. Therefore for these comments that were re-annotated, it lists NOT the average of all annotations but only the annotations from the respective author (G1-G3) who checked the comment.
For detailed description of the columns see Table 'Annotations'.

Query
 
CREATE VIEW ${name} AS
-- average the annotations for all comments, that were not annotated by G1-G3
SELECT 
	a.c_id,
	c.c_text,
	count(*) as nb_annotators,
	avg(a.dataset_id) as dataset_id,
	avg(strftime('%s',a.time_response)-strftime('%s', a.time_start)) as duration, -- mean annotation time in seconds
	avg(a.incomp) as incomp,
	avg(a.sentiment) as sentiment,
	avg(a.hate_speech) as hate_speech,
	avg(a.criminal_rel) as criminal_rel,
	avg(a.p_86) as p_86,
	avg(a.p_86a) as p_86a,
	avg(a.p_111) as p_111,
	avg(a.p_126) as p_126,
	avg(a.p_130) as p_130,
	avg(a.p_131) as p_131,
	avg(a.p_140) as p_140,
	avg(a.p_166) as p_166,
	avg(a.p_185) as p_185,
	avg(a.p_186) as p_186,
	avg(a.p_187) as p_187,
	avg(a.p_189) as p_189,
	avg(a.p_240) as p_240,
	avg(a.p_241) as p_241,
	-- split the expression in number of explicit and number of implicit
	count((SELECT a.expression WHERE a.expression="explicit")) as expression_explicit,
	count((SELECT a.expression WHERE a.expression="implicit")) as expression_implicit,
	avg(a.toxi) as toxi,
	-- split the aim in number of person, group, public
	count((SELECT a.target WHERE a.target="person")) as target_person,
	count((SELECT a.target WHERE a.target="group")) as target_group,
	count((SELECT a.target WHERE a.target="public")) as target_public,
	avg(a.discrim_job) as discrim_job,
	avg(a.discrim_attitude) as discrim_attitude,
	avg(a.discrim_engagement) as discrim_engagement,
	avg(a.discrim_sexIdent) as discrim_sexIdent,
	avg(a.discrim_characteristics) as discrim_characteristics,
	avg(a.discrim_nation) as discrim_nation,
	avg(a.discrim_religion) as discrim_religion,
	avg(a.discrim_socialStatus) as discrim_socialStatus,
	avg(a.discrim_worldview) as discrim_worldview,

	FROM Annotations as a 
	JOIN Comments as c ON a.c_id=c.c_id
	WHERE a.c_id NOT IN (SELECT t.c_id FROM Annotations as t WHERE t.name IN ("G1","G2","G3"))
	GROUP BY a.c_id
	
UNION ALL

View v_Voting

This view computes the averages of all annotations for every comment that has annotations. Furthermore it has the column 'nb_annotators' that states, how many times the comment was annotated.
For the categories expression and target the annotations are split into 2 and 3 columns respectively. E.g. for expression it counts how many times it was annotated as 'explicit' and how many times as 'implicit'.

Query
 
CREATE VIEW ${name} AS
SELECT

	c.c_text,
	count(*) as nb_annotators, -- Anzahl Annotatoren, die den Kommentar annotiert haben
	avg(a.dataset_id) as dataset_id,
	avg(strftime('%s',a.time_response)-strftime('%s', a.time_start)) as duration, -- mittlere Annotationszeit in s
	avg(a.incomp) as incomp,
	avg(a.sentiment) as sentiment,
	avg(a.hate_speech) as hate_speech,
	avg(a.criminal_rel) as criminal_rel,
	avg(a.p_86) as p_86,

	-- split the aim in number of person, group, public

	count((SELECT a.target WHERE a.target="group")) as target_group,
	count((SELECT a.target WHERE a.target="public")) as target_public,
	avg(a.discrim_job) as discrim_job,
	avg(a.discrim_attitude) as discrim_attitude,
	avg(a.discrim_engagement) as discrim_engagement,
	avg(a.discrim_sexIdent) as discrim_sexIdent,

View v_dupAnnotations

This view gives a list of comments that were annotated twice by the authors.
Every row gives the comment-ID of the twice annotated comment and the author who annotated this comment twice.

Query
 
CREATE VIEW ${name} as
	SELECT a.c_id, a.name FROM Annotations as a
	GROUP BY a.c_id, a.name
	HAVING COUNT(*) > 1
IdxField NameData TypeDescription
  c_id TEXT
  name TEXT
